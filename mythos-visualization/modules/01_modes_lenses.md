# Module 01 — Modes, Lenses, and Routing

Load this module when the always-load kernel needs finer routing, mode selection, or lens-pruning guidance.

## Mode selection details

Choose the smallest mode that preserves the user's intended outcome.

- **Ultra-Compact**: direct identity, triage, high-volume row, quick alt text.
- **Tiny**: caption or one-purpose answer with one or two cues.
- **Quick**: useful read without audit machinery.
- **Compressed Standard**: normal default for meaningful visual analysis.
- **Standard**: multi-part analysis, comparison, critique, extraction, reconstruction, implementation guidance.
- **Forensic**: high-stakes, disputed, compliance, source-to-render, pixel-level, accessibility, legal/medical/safety, or reproducibility work.
- **Corpus profile**: batch/folder workflows layered on D0–D4 coverage.
- **Release profile**: skill/model/report comparison, variants, score matrix, checksums, preserved behavior.

Do not escalate because an artifact is interesting; escalate only because missing evidence could change the answer.

## Dominant lens routing

Use exactly one dominant lens unless the user asks for multi-lens comparison.

1. Task/action interface → **UI/product/workflow**.
2. Encoded quantities → **chart/data**.
3. Spatial extent, route, layers, place → **map/spatial**.
4. Entities and relationships/process → **diagram/system**.
5. Real-world scene → **photo/scene**.
6. Biological/object identification → **creature/object**.
7. Style, symbolism, narrative, visual culture → **artwork/style**.
8. Caption-image joke or social reference → **meme/social**.
9. Non-representational form → **abstract/formal**.
10. Scientific, industrial, medical-looking imagery → **scientific/industrial**.
11. Source files or vector/programmatic visuals → **source-backed graphic**.
12. Sequence/video/interactive state over time → **motion/video**.
13. Folder/benchmark/mixed set → **mixed-corpus**.

## Supporting lenses

Use supporting lenses only when they add evidence: color/light, typography, geometry, source structure, motion, scene graph, accessibility risk, measurements, exact extraction, safety/claim boundary, or cultural context.

## Route confidence

Use route confidence for corpora, ambiguous artifacts, degraded files, or label/content conflict:

- **High**: observed/measured content and metadata agree.
- **Medium**: plausible route but weak, partial, or coarse evidence.
- **Low**: unreadable, tiny, ambiguous, contradictory, or metadata-led.

Compact form: `dominant lens; confidence; evidence used; conflict; evidence not verified`.

## De-escalation guard

If a section would not change identity, task, interpretation, risk, classification, reconstruction, or validation, omit it.
