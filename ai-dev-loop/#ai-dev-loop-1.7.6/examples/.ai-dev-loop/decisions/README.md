# Decision Records

Create decision records here only for durable product, risk, architecture, dependency, data, cost, migration, validation, packaging, or scope decisions.

Use a decision record when the choice should guide future R/K turns, releases, tests, documentation, or support work. Do not create one for routine edits or temporary task notes.

A good decision record includes:

- Decision and status.
- Context that forced the choice.
- Options considered.
- Chosen option and rationale.
- Consequences and trade-offs.
- Validation or enforcement mechanism.
- Revisit trigger.

Decision records should be cited from `status.md` when active.
