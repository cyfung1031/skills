# Context Notes

Create compact context handoff notes here when a loop is long, risky, likely to continue across sessions, or hard to reconstruct from the latest R/K record alone.

A good context note answers:

- What was the user trying to accomplish?
- Which files, docs, tests, scripts, examples, or package artifacts are in scope?
- Which R findings are still open, closed, objected to, or blocked?
- What validation was actually run, and what was skipped?
- Which durable decisions should the next role load from `decisions/`?
- What should the next role do first?

Context notes are not a substitute for `status.md`, R reviews, K responses, validation logs, or decision records. They are compression for continuity.
